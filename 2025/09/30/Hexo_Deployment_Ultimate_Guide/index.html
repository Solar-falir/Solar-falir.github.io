4. Hexo 文件目录功能深度解析 📂

  • _config.yml ⚙️
    • 功能：站点配置文件。这是 Hexo 博客的 全局配置文件，用于设置整个网站的标题、副标题、描述、URL、语言、时区、分页、文章链接格式、默认主题、部署方式等所有核心配置。
    • 原理：Hexo 在生成网站时会读取此文件，根据其中的配置来渲染页面、处理文章和部署。
    • 注意点 ⚠️：YAML 格式对缩进非常敏感，请务必使用 两个空格 而非 Tab 键进行缩进。错误的缩进会导致配置解析失败。
  • package.json 📦
    • 功能：Node.js 项目的配置文件。记录了项目的名称、版本、依赖包列表（dependencies）、脚本命令等信息。
    • 原理：npm install 命令会根据此文件安装依赖；npm run <script_name> 命令会执行其中定义的脚本。
  • scaffolds/ 📝
    • 功能：模板文件夹。存放文章、页面、草稿等内容的 模板文件。当您使用 hexo new <layout> <title> 命令创建新内容时，Hexo 会根据这里的模板生成文件。
    • 原理：这些模板定义了新创建内容的默认 Front-matter（元数据）结构。
  • source/ 📂
    • 功能：源文件文件夹。存放您博客的所有原始内容，包括文章、页面、图片等。这是您博客内容的“心脏”。
    • 原理：Hexo 会扫描此目录下的所有文件，并根据其类型和 Front-matter 进行处理。
    • 子目录：
      • _posts/ ✍️：存放您的所有博客文章（Markdown 格式）。
      • _data/ 📊：[关键] 存放主题可能需要的额外数据文件（如 .yml），这些数据会通过 site.data 变量在主题中访问。
  • themes/ 🎨
    • 功能：主题文件夹。存放您安装的所有 Hexo 主题。每个主题都有自己的子文件夹。
    • 原理：主题定义了您博客的视觉外观和布局。Hexo 会根据 _config.yml 中 theme 字段指定的名称，加载对应的文件夹作为当前主题。
  • public/ 🌐 (自动生成)
    • 功能：公共文件夹。hexo generate 命令会在这里生成所有静态文件。永远不要手动修改此文件夹。
    • 原理：这是 Hexo 静态站点生成的核心输出。
  • db.json 💾 (自动生成)
    • 功能：Hexo 的数据缓存文件，hexo clean 会删除它。
  • node_modules/ 🧩 (自动生成)
    • 功能：存放所有项目依赖的模块。不应提交到 Git。

3. 主题目录下的文件及文件夹功能深度解析 📂

每个 Hexo 主题都是一个独立的、功能完备的“皮肤包”。理解其内部结构，是进行个性化定制、解决显示问题、甚至开发自己功能的基础。我们将以 hexo-theme-researcher 为例，层层剖析。

  • _config.yml ⚙️

    • 功能：主题配置文件。这是主题专属的“控制面板”，独立于您博客根目录下的站点 _config.yml。它负责控制所有与主题外观和功能相关的选项。
    • 原理 🧠：当 Hexo 渲染页面时，会首先读取站点的 _config.yml，然后再读取主题的 _config.yml。主题模板（.ejs 文件）通过 theme. 这个变量来访问此文件中的配置项（例如 theme.avatar, theme.menu）。
    • researcher 主题中的例子：
      • menu: 控制导航栏的显示内容和链接。
      • avatar: 定义头像图片的路径。
      • academic_profiles: 配置您的 Google Scholar、GitHub 等学术社交链接。
      • daisyui: researcher 主题使用 DaisyUI 库，这里允许您配置启用的颜色主题。

  • layout/ 🖼️

    • 功能：布局与模板文件夹。这是主题的“骨架”，决定了您网站的 HTML 结构。该文件夹下的 .ejs 文件（EJS 是一种模板引擎）是最终生成 HTML 页面的“模具”。
    • 原理 🧠：当 Hexo 生成一个页面时，它会首先根据该页面的 Front-matter layout 属性，在此文件夹中寻找同名的 .ejs 文件。如果找不到，它会使用默认的 page.ejs 或 post.ejs。这些模具最终通常会嵌套在 layout.ejs 这个主布局文件中。
    • researcher 主题中的例子：
      • layout.ejs：最顶层的母版页，定义了所有页面共享的 <html>, <head>, <body> 结构。
      • partials/：存放“局部”或“可重用”的模板片段，如 header.ejs, sidebar.ejs, footer.ejs。
      • index.ejs: 博客主页（文章列表）的布局。
      • post.ejs: 单篇文章页面的布局。
      • publications.ejs, projects.ejs, cv.ejs: 这些都是我们为特定页面创建的自定义布局。

  • source/ 📦

    • 功能：主题自身的前端静态资源文件夹。存放主题正常工作所需的 CSS、JavaScript、图片、字体等文件。
    • 原理 🧠：在 hexo generate 过程中，此文件夹下的所有内容都会被原封不动地复制到最终的 public 目录中，供浏览器加载。
    • researcher 主题中的例子：
      • css/ 💅: 存放主题的样式表，如 style.css 和 tailwind.css。
      • js/ 📜: 存放主题所需的 JavaScript 脚本。
      • images/ 🖼️: 存放主题自带的图片，例如默认的头像 avatar.jpg。

  • languages/ 🗣️

    • 功能：多语言文件夹。用于实现主题的国际化 (i18n)。
    • 原理 🧠：该目录下会存放 en.yml, zh-CN.yml 等语言文件，定义了主题中硬编码文本（如“阅读更多”）在不同语言下的翻译。Hexo 会根据您在站点 _config.yml 中设置的 language 字段，自动加载对应的语言文件。

  • scripts/ 📜

    • 功能：主题自定义脚本文件夹。存放 Node.js 脚本，用于在 Hexo 生成期间扩展其功能。
    • 原理 🧠：Hexo 在启动时会自动加载此文件夹下的所有 .js 文件。您可以在这里编写脚本来注册自定义的“标签插件”（Tag Plugins）或“辅助函数”（Helpers）。

  • demo_data/ 📊

    • 功能：示例数据文件夹。主题作者提供的 .yml 格式的示例数据，用于展示主题功能和数据结构。
    • 原理 🧠：这些文件本身在主题中不起任何作用，它们的存在是为了作为“模板”，告诉用户应该在自己博客的 source/_data/ 目录下创建什么样结构的数据文件。

4. 实战问题解析（一）：修复主题渲染错误 🐛➡️✅

  • 问题现象：运行 hexo server 后，页面显示 TypeError: Cannot read properties of undefined (reading 'filter')。
  • 问题根源：主题模板需要 source/_data/ 目录下的数据文件（如 publications.yml）来渲染页面，但这些文件在您的项目中尚不存在，导致模板访问了 undefined 而报错。
  • 解决方案：
    1. 提供数据：将主题作者提供的示例数据复制到您的项目中。
    2. 提供布局：为主题菜单中定义的每个页面创建对应的文件，并指定正确的 layout。
      • 创建页面: hexo new page publications && hexo new page projects && ...
      • 修改布局: 打开 source/publications/index.md 等文件，修改其 Front-matter。

V. 部署与“创世”备份

1. 核心概念：git push vs hexo deploy

  • git push: 推送源码。这是将你的博客“原材料”（.md 文件、配置文件等）备份到你的源码仓库。这不会让你的网站上线。
  • hexo deploy: 部署网站。这是运行 hexo g 将网站生成为静态文件，然后将这些最终产物推送到你的部署仓库，这才会让你的网站更新上线。

2. 创建 GitHub 仓库

  • 源码仓库：创建一个私有仓库，如 MyHexoBlog-source。
  • 部署仓库：创建一个公开仓库，严格命名为 <Your-Username>.github.io。

VI. 个性化定制实战

6.1 场景一：修改个人信息

  • 目标：将博客的作者、头像、社交链接等替换为您自己的信息。
  • 位置：themes/researcher/_config.yml (主题配置文件)。
  • 操作：打开此文件，根据其中的注释，逐一修改 avatar (替换 themes/researcher/source/images/avatar.jpg 图片文件), about, academic_profiles, contact 等字段。

6.2 场景二：管理动态内容 (如 Publications)

  • 目标：在“出版物”页面新增一条您自己的记录。
  • 位置：source/_data/publications.yml。
  • 原理:

    此 YAML 文件是一个结构化数据列表。主题的 layout/publications.ejs 模板文件内部有 EJS 逻辑（类似于 site.data.publications.forEach(...)），它会遍历此列表中的每一个条目，并将每个条目的 title, authors 等属性填充到预设的 HTML 结构中，从而动态生成您看到的列表。

  • 操作：模仿该文件中已有的条目格式（注意 YAML 的 - 和缩进），复制并修改为您自己的内容。

6.3 实战场景三：添加全新的“作品集(Portfolio)”页面

  • 目标：从零开始，为您的博客创建一个展示图片和描述的独立作品集页面。
  • 操作流程：
    1. 创建数据源：在 source/_data/ 目录下新建 portfolio.yml 文件，并添加您的项目数据。
    2. 创建布局文件：在 themes/researcher/layout/ 目录下，新建 portfolio.ejs 文件，并写入 EJS 代码以遍历和显示 site.data.portfolio 中的数据。
    3. 创建页面文件：hexo new page portfolio。
    4. 修改页面 Front-matter：打开 source/portfolio/index.md，设置 layout: portfolio。
    5. 添加菜单项：打开 themes/researcher/_config.yml，在 menu 部分添加新页面的链接。
    6. 添加图片资源：将作品集图片放入 themes/researcher/source/images/portfolio/ 目录。
    7. 预览：运行 hexo server 查看效果。

VII. 日常维护与未来规划

7.1 黄金工作流 (日常)

  1. 本地预览：hexo server，在 http://localhost:4000 实时查看修改。
  2. 内容创作：修改或撰写 .md 文件。
  3. [推荐] 备份源码：git add . && git commit -m "Update: ..." && git push
  4. 发布网站：hexo clean && hexo deploy --generate

7.2 维护案例一：更新简历

  • 场景：您有了一份新的 PDF 简历需要替换旧的。
  • 操作：将新 PDF 放入 source/files/，然后修改 themes/researcher/_config.yml 中的 cv.download_link 路径，最后重新部署。

7.3 维护案例二：更换域名

  • 场景：您购买了个人域名 www.my-awesome-blog.com。
  • 操作：
    1. 在域名提供商处，将域名 CNAME 解析到 <Your-Username>.github.io。
    2. 在 source/ 目录下创建 CNAME 文件，内容为您的新域名。
    3. 修改站点 _config.yml 的 url 为新域名。
    4. 重新部署。

VIII. 高级技巧与拓展

8.1 高级 Git 技巧：使用分支进行无风险修改

  • 场景：想对主题进行一次大的修改，但不确定效果，也不想污染您干净的 master 分支。
  • 操作：
    1. 创建并切换到新分支：git checkout -b redesign-theme
    2. 进行修改和提交：在新分支上任意修改和 git commit。
    3. 切换回主分支：git checkout master (工作区将恢复原样)。
    4. 合并分支：如果对修改满意，git merge redesign-theme。

8.2 高级 Hexo 技巧：Front-matter 默认值

  • 场景：所有文章都有固定的分类或作者，每次都重复写很麻烦。
  • 操作：在站点 _config.yml 中添加 front_matter 配置来设置默认值。

8.3 自动化部署 (CI/CD) 概念

  • 场景：希望实现“只要将源码推送到 GitHub，网站就自动部署”。
  • 原理：使用 GitHub Actions 在云端服务器上自动执行 hexo deploy 命令。
  • 实现：在源码仓库中创建 .github/workflows/deploy.yml 文件并编写工作流脚本。这是一个高级主题，可搜索“Hexo GitHub Actions”获取详细教程。
